No student name, roll number, email, section, batch, or IP is ever stored or passed to any model. Gender and race columns are dropped at ingestion — not used anywhere in the pipeline. All identifiers stripped at collection. Feedback is aggregated only at faculty × course × semester level. Groups with fewer than 5 responses are suppressed from all outputs.

Sentiment analysis
Baseline: VADER (no training)
Model: fine-tuned BERT
Train on RMP → fine-tune on Indian data.
Output: Positive / Neutral / Negative + confidence score.
Step 1 — VADER baseline: Run VADER on all comments. No training needed. Fast benchmark. Use SentimentIntensityAnalyzer().polarity_scores(text)['compound'].

Step 2 — BERT fine-tune: Label RMP data using sentiment_label column (already created in Step 6). Fine-tune bert-base-uncased on RMP comments. Use HuggingFace Trainer API. Train/val split 80/20.

Step 3 — Transfer to Indian data: Few-shot fine-tune on Indian survey comments (even 200 labelled examples improves significantly). Output: sentiment label + confidence score per comment.

Confidence threshold: Only use predictions with confidence >0.70. Flag uncertain ones separately.
